# ☕ Barrista Admin

Админ-панель для управления рецептами кофе и напитков.

## Возможности

- ✅ Полное управление напитками (CRUD операции)
- ✅ Удобная форма для добавления/редактирования напитков
- ✅ Управление ингредиентами, инструкциями и категориями
- ✅ Перевод напитков через API на несколько языков
- ✅ Экспорт данных в JSON формат
- ✅ База данных **SQLite** (файл `server/data/barrista.db`) — удобно экспортировать и бэкапить
- ✅ Кнопка «Скачать базу SQLite» — скачивание файла БД из админки
- ✅ Современный и адаптивный дизайн

## Установка

1. Установите зависимости:
```bash
npm install
```

2. Запустите приложение (одной командой поднимаются и API, и фронт):
```bash
npm run dev
```

3. Откройте в браузере:
   - фронт: **http://localhost:5173**
   - API: **http://localhost:3001**

## Структура данных

### Drink
```typescript
{
  id: string;
  title: string;
  ingredients: Ingredient[];
  instructions: string[];
  dishId: string;
  portionsAmount: number;
  categories: string[];
}
```

### Ingredient
```typescript
{
  title: string;
  volume: string;
}
```

## Использование

### Добавление напитка
1. Нажмите кнопку "Добавить напиток"
2. Заполните все поля формы
3. Добавьте ингредиенты, инструкции и категории
4. Нажмите "Сохранить"

### Переводы
1. Добавьте коды языков в поле "Код языка" (например: `en`, `es`, `fr`)
2. Нажмите "Добавить язык"
3. При экспорте используйте кнопку "Экспорт с переводами"

### Экспорт
- **Экспортировать JSON** — скачивает JSON со всеми напитками
- **Экспорт с переводами** — JSON с переводами на выбранные языки
- **Скачать базу SQLite** — скачивает файл `barrista.db` (можно копировать, открыть в DB-клиенте или подставить в другое окружение)

## Настройка API перевода

В файле `src/services/translation.ts` замените `TRANSLATION_API_URL` на адрес вашего API и обновите функцию `translateText` под формат ответа вашего API.

Пример API должен принимать:
```json
{
  "text": "Текст для перевода",
  "targetLang": "en"
}
```

И возвращать:
```json
{
  "translatedText": "Translated text"
}
```

Для тестирования можно использовать функцию `mockTranslateDrink` вместо `translateDrink` в компоненте `ExportButton.tsx`.

## Сборка и запуск одной точкой входа

```bash
npm run build
npm run start
```

Откройте **http://localhost:3001** — и админка, и API работают с одного порта. База SQLite: `server/data/barrista.db` (файл можно копировать и открывать в любом клиенте SQLite/MySQL-совместимом).
